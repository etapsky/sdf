# SDF — OS Integration

> Registers `.sdf` as a known file type on macOS and Windows.

[![License: BUSL-1.1](https://img.shields.io/badge/License-BUSL--1.1-blue.svg)](../../LICENSE)

Part of the [Etapsky SDF](https://github.com/etapsky/sdf) monorepo · `tooling/os-integration/`

---

## What it does

Without OS integration, `.sdf` files appear as unknown ZIP archives. With it:

| | macOS | Windows |
|---|---|---|
| Finder / Explorer icon | Custom SDF icon | Custom SDF icon |
| Space key preview | `visual.pdf` rendered via Quick Look | — |
| Double-click | Opens in SDF Reader | Opens in SDF Reader |
| Right-click menu | — | Inspect with SDF CLI · Validate SDF |
| MIME type | `application/vnd.sdf` registered | `application/vnd.sdf` registered |

## macOS

### Requirements

- macOS 12+
- Xcode 15+ (to build the Quick Look plugin)
- Optional: SDF Reader at `/Applications/SDF Reader.app`
- Optional: `duti` for file association (`brew install duti`)
- Optional: Pillow for icon generation (`pip install Pillow`)

### Quick Look plugin

The plugin extracts `visual.pdf` from the `.sdf` ZIP archive and passes it to Quick Look for rendering. If `visual.pdf` is unavailable, it falls back to an HTML preview of `meta.json`.

**Build:**

```bash
cd tooling/os-integration/macos
xcodebuild -project SDFQuickLook.xcodeproj -target SDFQuickLook -configuration Release
```

**Install:**

```bash
cd tooling/os-integration/macos
chmod +x install-macos.sh
./install-macos.sh
```

**Manual install** (pre-built `.qlgenerator` bundle):

```bash
cd tooling/os-integration/macos
cp -r build/SDFQuickLook.qlgenerator ~/Library/QuickLook/
qlmanage -r
```

**Test** (from repo root):

```bash
# Generate a sample .sdf
cd spec/poc && npm run build:invoice

# Test Quick Look preview
qlmanage -p spec/poc/output/invoice.sdf
```

### Icon generation

Icons (`SDFDocument.icns` for macOS, `SDFDocument.ico` for Windows) are generated by `generate-icon.py`:

```bash
cd tooling/os-integration/macos
pip install Pillow
python3 generate-icon.py
```

Output: `SDFDocument.icns` (→ copy into `.qlgenerator` bundle `Resources/`), `SDFDocument.ico` (→ use for Windows registry / app resources).

### Plugin source

| File | Description |
|------|-------------|
| `Info.plist` | Bundle metadata, UTI declaration, document type registration |
| `GeneratePreviewForURL.swift` | Preview generator — extracts `visual.pdf` and returns it to Quick Look |
| `GenerateThumbnailForURL.swift` | Thumbnail generator — shows first page of `visual.pdf` in Finder |
| `SDFQuickLook.entitlements` | Sandbox entitlements for the Quick Look XPC process |
| `install-macos.sh` | Install script — copies plugin, resets Quick Look daemon, registers UTI |

## Windows

### Requirements

- Windows 10 / 11
- PowerShell 5+ (run as Administrator)
- Node.js + npm (for right-click CLI menu items)

### Install

```powershell
# Run PowerShell as Administrator (from repo root)
cd tooling/os-integration/windows
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\install-windows.ps1
```

**Options:**

| Option | Description |
|--------|-------------|
| *(none)* | Apply registry entries, verify Node.js/npx, refresh Explorer |
| `-RegistryOnly` | Only apply registry entries, skip dependency checks |
| `-Uninstall` | Remove all SDF registrations, refresh Explorer |

Or double-click `sdf-association.reg` and accept the UAC prompt.

### What the registry entries do

- `HKCR\.sdf` — maps `.sdf` extension to `SDFDocument` ProgID
- `HKCR\SDFDocument` — friendly name, icon, open/inspect/validate shell commands
- `HKLM\SOFTWARE\Classes\MIME\...` — MIME type registration
- Right-click "Inspect with SDF CLI" — runs `npx @etapsky/sdf-cli inspect`
- Right-click "Validate SDF" — runs `npx @etapsky/sdf-cli validate`

### Uninstall

```powershell
cd tooling/os-integration/windows
.\install-windows.ps1 -Uninstall
```

Or double-click `undo-sdf-association.reg`.

### Test

After installing, verify the registry setup (no admin required):

```powershell
cd tooling/os-integration/windows
.\test-windows.ps1
```

### Files

| File | Description |
|------|-------------|
| `sdf-association.reg` | Registry entries — extension, MIME, shell commands |
| `undo-sdf-association.reg` | Removes all SDF registry entries |
| `install-windows.ps1` | PowerShell installer — applies registry, verifies dependencies, refreshes shell |
| `test-windows.ps1` | Verifies that registry integration is correctly applied |

## MIME type

Both platforms register `application/vnd.sdf` as the canonical MIME type for `.sdf` files. This enables:

- Correct Content-Type headers when serving `.sdf` files from `sdf-server`
- Browser download dialogs showing the correct file type
- Email clients recognising `.sdf` attachments
